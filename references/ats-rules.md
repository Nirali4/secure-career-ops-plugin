# ATS Resume Compliance Rules

These rules ensure your resume passes Applicant Tracking System parsers.
Every resume generated by this plugin follows these rules.

## Layout

- Single column ONLY. No sidebars, no two-column designs.
- No text boxes, shapes, or floating elements.
- No headers or footers (ATS often can't read them). Put your name and
  contact info in the body, not the header.
- Maximum 2 pages. 1 page for early career (< 5 years).

## Formatting

- Standard fonts: Arial, Calibri, Cambria, Georgia, Times New Roman,
  or system-default sans-serif.
- Body text: 10-12pt. Name: 14-16pt. Section headers: 12-14pt bold.
- Margins: 0.5 to 1 inch on all sides.
- Use bold and italic sparingly. No underlining (confused with hyperlinks).
- Consistent date format throughout: "Jan 2020 - Present" or
  "January 2020 - Present" (pick one, stick with it).

## Structure

Use standard section headers EXACTLY as written:
- "Experience" (not "Professional Journey" or "Career History")
- "Education" (not "Academic Background")
- "Skills" (not "Competencies" or "Toolkit")
- "Certifications" (not "Credentials")
- "Summary" or "Professional Summary" (if included)

For each job entry:
- Company name, title, and dates in clear visual hierarchy
- Location on same line as company (optional)
- 3-5 bullet points per role

## Content

- All text must be real text (selectable, searchable). No text-in-images.
- No tables nested inside tables. Simple tables OK for skills grid.
- No images, logos, icons, or graphics of any kind.
- Bullet points: use standard bullet or hyphen. Not fancy symbols.
- Links: include LinkedIn URL as plain text, not hyperlink-only element.
- Include both acronym and full form: "Search Engine Optimization (SEO)"

## Keyword Strategy

- Mirror exact phrases from the JD. If the JD says "project management,"
  your resume says "project management," not "programme management" or "PM."
- Place the most important keywords in the first third of the resume
  (Summary and first role).
- Skills section lists JD keywords first, then additional skills.
- Each bullet should contain at least one relevant keyword where natural.

## File Output

- HTML that prints to clean PDF via browser Cmd+P / Ctrl+P
- Filename convention: `{FirstName}-{LastName}-{Company}-{Role}.html`
- Include `@media print` CSS for clean print output
- `@page` rules for correct paper size (Letter for US, A4 for international)
- No JavaScript in the HTML (breaks some ATS parsers)
